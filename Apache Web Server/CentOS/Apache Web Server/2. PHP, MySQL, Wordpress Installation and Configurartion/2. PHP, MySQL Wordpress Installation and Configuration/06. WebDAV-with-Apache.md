

# Apache WebDAV + Directory Listing

WebDAV = HTTP ke through **remote file upload, edit, delete, move** karne ka system.
Apache me `mod_dav` + `mod_dav_fs` se chalta hai.

---

## 1. Apache Install & Verify

### Check installed or not

```bash
rpm -qa | grep httpd
```

### Install Apache (RHEL/CentOS/Alma/Rocky)

```bash
yum install httpd httpd-tools -y
```

(`httpd-tools` important hai → htpasswd ke liye)

---

## 2. Required Modules Enable 

### Check modules

```bash
httpd -M | grep dav
```

### Expected output

```bash
dav_module (shared)
dav_fs_module (shared)
```

Agar nahi dikhe:

### Enable modules

```bash
yum install mod_dav_fs -y
```

Restart:

```bash
systemctl restart httpd
```

---

## 3. Apache Service Start + Enable

```bash
systemctl enable httpd.service
systemctl start httpd.service
systemctl status httpd.service
```

Green "active (running)" hona chahiye.

---

## 4. WebDAV Directory Create

### Directory create

```bash
mkdir -p /var/www/html/webdav
```

### Ownership

```bash
chown -R apache:apache /var/www/html/webdav
```

### Permissions

```bash
chmod -R 775 /var/www/html/webdav
```

(777 mat use karo production me)

---

## 5. DIRECTORY LISTING ENABLE (tumhara main missing point)

Apache directory listing ke liye **mod_autoindex + Options Indexes** zaroori hai.

### Confirm autoindex

```bash
httpd -M | grep autoindex
```

Expected:

```bash
autoindex_module (shared)
```

## 7. Apache Config (MAIN STEP)

### Open config

```bash
vim /etc/httpd/conf/httpd.conf
```

### ADD THIS FULL BLOCK (corrected + complete)

```apache
Alias /webdav /var/www/html/webdav

<Directory /var/www/html/webdav>
    DAV On

    Options Indexes FollowSymLinks
    AllowOverride None

    Require all granted
</Directory>
```

### Why?

* DAV On → WebDAV enable
* Indexes → directory listing ON
* Require all granted → access allow
* Alias → clean URL

## 9. SELinux Fix (disable mat karo)

### Check

```bash
sestatus
```
- Enable SElinux
```
vim /etc/selinux/config
```
```
SELINUX=enforcing
```
OR
```
SELINUX=permissive
```
### Correct way

```bash
chcon -R -t httpd_sys_rw_content_t /var/www/html/webdav
```

(write permission ke liye rw use karo)

## 12. Secure with Basic Authentication

```bash

```

Add more:

```bash
htpasswd /etc/httpd/htpasswd admin
```

```bash
htpasswd /etc/httpd/htpasswd root
```

```bash
htpasswd /etc/httpd/htpasswd user
```

---

## 13. Update Config with Auth

Again edit:

```bash
vim /etc/httpd/conf/httpd.conf
```

Replace block with:

```apache
Alias /webdav /var/www/html/webdav

<Directory /var/www/html/webdav>
    DAV On

    Options Indexes FollowSymLinks
    AllowOverride None

    AuthType Basic
    AuthName "webdav"
    AuthUserFile /etc/httpd/htpasswd
    Require valid-user
</Directory>
```

Restart:

```bash
systemctl restart httpd.service
```

## Final Reality Check (no sugarcoat)

Agar directory listing abhi bhi nahi dikhti:

* mod_autoindex missing
* Options Indexes missing
* Apache restart bhool gaye
* SELinux block kar raha
* Wrong path in Alias

In 90% cases problem **Indexes option missing hota hai**.
